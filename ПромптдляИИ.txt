Ты — senior full-stack разработчик и архитектор. Тебе нужно создать полноценный enterprise-сайт MathLife с нуля. Проект должен быть масштабируемым, модульным, содержать более 150 файлов, современный стек и впечатляющий дизайн с 3D-элементами. Сайт представляет собой платформу математических калькуляторов, организованных по категориям. Требования: минимум 50 тщательно проработанных калькуляторов, 5+ цветовых тем, большое разнообразие шрифтов (возможность выбора), множество интерактивных графиков для наглядности, а также реализация большинства дополнительных идей (кроме голосового ввода).
1. Общее описание проекта

MathLife — профессиональный веб-портал для выполнения различных математических вычислений. Сайт должен выглядеть как дорогой enterprise-продукт (премиум качество). Основная аудитория: студенты, инженеры, финансисты, преподаватели. Главная страница — дашборд с популярными калькуляторами, последними вычислениями и 3D-сценой. Далее — множество страниц с калькуляторами, сгруппированных по категориям (арифметика, алгебра, геометрия, тригонометрия, статистика, финансы, инженерные, конвертеры и т.д.).
2. Технологический стек (использовать современные инструменты)

    Frontend: React 18+ (Next.js 15+ с App Router для SSR/SSG — обязательно, чтобы обеспечить быструю загрузку и SEO для множества страниц).

    Язык: TypeScript (строгая типизация, интерфейсы для всех компонентов и пропсов).

    Стили: Tailwind CSS + CSS Modules для сложных компонентов. Framer Motion для анимаций.

    3D: Three.js (react-three-fiber/drei для удобной интеграции с React).

    Графики: Библиотека для визуализации данных (например, Recharts, Visx, D3.js, Plotly.js или ECharts). Должна поддерживать различные типы графиков: линейные, столбчатые, круговые, точечные, 3D-поверхности, тепловые карты.

    Управление состоянием: Zustand или Redux Toolkit (для глобальных состояний: история вычислений, настройки темы/шрифта, избранное).

    Формулы: KaTeX или MathJax для отображения математических выражений.

    Тестирование: (опционально) добавить примеры тестов (Jest, React Testing Library), чтобы показать культуру enterprise.

    Инструменты: ESLint, Prettier, Husky (можно упомянуть в структуре).

    Маршрутизация: Next.js App Router с динамическими маршрутами для каждого калькулятора (например, /calculator/[slug]).

    i18n: next-i18next для поддержки русского и английского (и, возможно, других языков).

    UI-библиотеки: Можно использовать shadcn/ui или radix-ui как основу, но кастомизировать под дизайн.

3. Архитектура и структура файлов (более 150 файлов)

Проект должен содержать более 150 файлов (не считая node_modules и конфигурационных). Структура должна быть модульной и масштабируемой. Основные принципы:

    Каждый калькулятор — отдельная папка в components/calculators/ со своим набором файлов (компонент, стили, типы, тесты, утилиты).

    Для каждого типа графика — отдельные компоненты в components/charts/.

    Утилиты для математических вычислений разбиты по тематическим файлам (arithmetic.ts, algebra.ts, statistics.ts и т.д.).

    Константы, типы, хуки, контексты — в соответствующих папках.

    Возможна организация по фичам (feature-based) на усмотрение разработчика, главное — обеспечить читаемость и лёгкость добавления новых калькуляторов.

4. Дизайн и UX

    Темы: реализовать минимум 5 цветовых тем (например: светлая, тёмная, синяя неон, зелёная минималистичная, фиолетовая премиум, оранжевая энергичная). Переключение тем должно быть плавным, через CSS-переменные.

    Шрифты: предоставить пользователю возможность выбора шрифта для интерфейса из большого списка (минимум 10–15 вариантов, включая Inter, Roboto, Open Sans, Lato, Montserrat, Playfair Display, Source Code Pro и др.). Шрифты можно загружать динамически через Google Fonts или локально. Выбранный шрифт должен применяться ко всему интерфейсу (с возможностью сохранять выбор).

    Главная страница: встречает пользователя красивой 3D-сценой (например, вращающийся тор или волны из частиц, символизирующие числа). Под сценой — сетка из карточек-категорий калькуляторов. Также строка поиска с автодополнением (интеграция с идеей «умного поиска»).

    Цветовая гамма: каждая тема задаёт свои цвета, но общий стиль — современный, стеклянные эффекты (glassmorphism), градиенты, неоновые акценты.

    Адаптивность: десктоп (основной), планшеты, мобильные устройства.

    Микроанимации: плавные переходы между страницами, появление элементов, hover-эффекты, анимация при переключении тем/шрифтов.

    Страница калькулятора: для каждого калькулятора отдельная страница с подробным описанием, формулами (в LaTeX), полями ввода, кнопкой «Рассчитать» (не «=»), результатом, историей вычислений именно для этого типа, возможностью добавить в избранное.

5. Функциональность: 50+ проработанных калькуляторов

Необходимо создать минимум 50 различных калькуляторов, каждый из которых должен быть тщательно проработан (не просто "2+2", а полнофункциональный инструмент). Калькуляторы должны покрывать все основные области математики и приложений. Организуй их по категориям (категории будут отображаться в навигации). Каждый калькулятор должен иметь:

    Удобный интерфейс ввода (поля, кнопки, слайдеры, переключатели).

    Кнопка «Рассчитать» для запуска вычислений (возможен также автоматический пересчёт при изменении параметров, но основное действие — по кнопке).

    Показ результата с возможностью копирования.

    Пояснения, подсказки.

    Возможность использовать историю (общая история или контекстная).

    Валидация ввода.

    Для сложных вычислений (интегралы, производные) — либо численные методы, либо символьные (можно использовать math.js или подобные библиотеки).

Примерный список категорий и калькуляторов (нужно минимум 50, можно расширить этот список, добавляя вариации и специализированные калькуляторы). В качестве ориентира можно использовать предыдущие категории (базовые, научные, алгебра, матрицы, геометрия, тригонометрия, исчисление, статистика, финансы, конвертеры, специализированные), но их количество должно быть не менее 50.
6. Визуализация: ОЧЕНЬ много разнообразных графиков

Чтобы пользователь мог видеть данные наглядно, для каждого подходящего калькулятора необходимо добавить соответствующие графики. Требования:

    Графический калькулятор: построение 2D- и 3D-графиков функций (одной и двух переменных), возможность масштабирования, панорамирования, отображения нескольких кривых.

    Статистические калькуляторы: гистограммы, ящики с усами (box plots), круговые диаграммы, линейные графики распределений.

    Финансовые калькуляторы: графики платежей, накоплений, диаграммы cash flow.

    Матричные калькуляторы: визуализация матриц как тепловых карт.

    Геометрические калькуляторы: отображение фигур на плоскости или в пространстве (с использованием Three.js).

    Калькулятор функций: отображение производной/интеграла графически.

    Конвертеры: возможно, отображение шкал или сравнительных диаграмм.

Общая идея: сделать платформу максимально визуально ориентированной, чтобы пользователь не только получал числа, но и видел их интерпретацию.
7. Дополнительные возможности (enterprise-уровень)

    История вычислений: сохранять последние 20 операций (глобально или для каждого типа калькулятора) с возможностью повторного использования. История должна быть доступна в боковой панели.

    Избранное: пользователь может добавлять калькуляторы в избранное для быстрого доступа. Избранные калькуляторы отображаются в отдельном разделе на главной.

    Экспорт результатов: кнопка "Экспорт в PDF" или "Копировать в буфер обмена" (отправка на почту не требуется).

    Сохранение состояния: при перезагрузке страницы возвращаться к последнему открытому калькулятору.

    Многоязычность: реализовать поддержку русского и английского (i18n). Переключение языков в шапке.

    SEO: для Next.js продумать метатеги для каждой страницы, генерация sitemap.xml.

    Производительность: ленивая загрузка компонентов калькуляторов (React.lazy), оптимизация Three.js (использовать @react-three/drei для упрощения).

    PWA: можно добавить возможность установки как приложения (манифест, service worker).

10. Важные замечания

    Учитывай, что это enterprise-проект, поэтому избегай «магии» и хардкода. Выноси конфигурацию в отдельные файлы (constants/).

    Для 3D используй легкие модели либо процедурную генерацию, чтобы сайт не тормозил.

    Для стольких калькуляторов важно продумать производительность: ленивую загрузку, виртуализацию списков (если много данных).

    Убедись, что все калькуляторы действительно функциональны и протестированы (хотя бы базово).
